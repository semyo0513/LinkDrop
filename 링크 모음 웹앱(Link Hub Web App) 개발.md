# 📌 링크 모음 웹앱(Link Hub Web App) 개발 계획서

## 1. 프로젝트 개요
* **목표:** 자주 사용하는 학급 경영 및 교육용 웹 링크를 한곳에 모아두고, 크롬 새 탭(배경화면)처럼 직관적으로 접근할 수 있는 웹앱 구축.
* **주요 기능:**
  * 검색창을 통한 링크 빠른 검색 및 필터링
  * 스크롤 시 전체 링크 아이콘(그리드 형태) 표시
  * 아이콘 클릭 시 해당 링크 새 창으로 이동
  * 아이콘 옆 '설명 버튼(i)' 클릭 시 툴팁 또는 모달로 링크 설명 표시
* **기술 스택:**
  * **Frontend:** HTML5, CSS3, Vanilla JavaScript (배포: GitHub Pages)
  * **Backend (API):** Google Apps Script (GAS)
  * **Database:** Google Sheets

## 2. 데이터베이스 (Google Sheets) 구조
구글 시트의 1행은 헤더로 사용하며, 2행부터 데이터를 입력합니다.

| 범주 | 내용 | 링크 | 링크 설명 |
| :--- | :--- | :--- | :--- |
| 2.학급경영 | 모두의 선거 | http://joo.is/모두의선거 | 온라인 선거를 준비할 수 있습니다. 학번과 초기비밀번호 입력 후... |
| (예시)수업혁신 | 패들렛 | https://padlet.com/ | 실시간 협업 및 아이디어 공유 게시판 |

## 3. UI/UX 디자인 방향 (Chrome New Tab 스타일)
* **배경:** 깔끔한 단색 또는 자연 풍경 이미지 (사용자 커스텀 또는 랜덤 이미지 적용 고려).
* **상단 (Header):** 중앙 정렬된 넓은 검색창(Search Bar). 검색어 입력 시 즉각적인 필터링(Live Search) 적용.
* **중앙~하단 (Body):** 
  * 아이콘들이 바둑판(Grid) 배열로 정렬.
  * 하단으로 스크롤 시 전체 목록 자연스럽게 노출.
* **아이콘 컴포넌트 구조:**
  * **메인 영역:** 웹사이트 파비콘(Favicon) 또는 범주별 기본 아이콘 + '내용(타이틀)' 텍스트. 클릭 시 `target="_blank"`로 링크 이동.
  * **보조 버튼:** 아이콘 우측 상단이나 하단에 작게 '?' 또는 'i' 아이콘 배치. 클릭 시 '링크 설명' 텍스트 팝업(툴팁).

## 4. 단계별 개발 구현 계획

### Phase 1: 데이터베이스 및 API 구축 (Google Apps Script)
1. **구글 시트 생성 및 데이터 입력:** 위 스키마에 맞추어 기본 데이터(모두의 선거 등) 작성.
2. **Apps Script 작성:**
   * 시트 데이터를 읽어와 JSON 형태로 반환하는 `doGet(e)` 함수 작성.
   * CORS (Cross-Origin Resource Sharing) 문제를 방지하기 위해 `ContentService.createTextOutput()`에 JSONP 방식이나 적절한 헤더 적용.
3. **웹앱 배포:** GAS를 '모든 사용자(익명 포함)'가 접근할 수 있는 웹앱으로 배포하고 `API URL` 확보.

### Phase 2: 프론트엔드 기본 구조 및 UI 개발
1. **GitHub 저장소 생성:** `link-hub-webapp` 등의 이름으로 레포지토리 생성.
2. **기본 마크업 (index.html):** 검색창 영역과 링크 아이콘이 들어갈 컨테이너(`<div id="link-container"></div>`) 작성.
3. **스타일링 (style.css):**
   * CSS Flexbox 또는 Grid를 활용해 크롬 배경화면 스타일의 레이아웃 구성.
   * 모바일 및 태블릿 접속을 고려한 반응형(Responsive) 디자인 적용.

### Phase 3: 핵심 기능 구현 (JavaScript)
1. **데이터 Fetching:**
   * `fetch()` API를 사용하여 Phase 1에서 확보한 GAS API URL에서 JSON 데이터 호출.
2. **UI 렌더링:**
   * 불러온 데이터를 기반으로 DOM 요소를 동적으로 생성하여 화면에 뿌려줌.
   * 각 링크의 파비콘을 자동으로 가져오는 서비스(예: `https://www.google.com/s2/favicons?domain={URL}`) 연동.
3. **검색 기능 구현:**
   * 검색창에 `input` 이벤트 리스너를 달아, 입력값과 시트의 '내용' 또는 '범주'가 일치하는 항목만 화면에 남기도록 필터링 함수 작성.
4. **설명 보기(툴팁) 기능:**
   * 설명 버튼 클릭 이벤트 구현 (CSS 토글 클래스 추가 방식 또는 모달 창 띄우기).

### Phase 4: 테스트 및 배포 (GitHub Pages)
1. **로컬 테스트:** 검색, 스크롤, 링크 이동, 설명 팝업이 정상 작동하는지 확인.
2. **GitHub 연동 및 Push:** 완성된 코드를 GitHub 레포지토리에 업로드.
3. **GitHub Pages 활성화:** 저장소 설정(Settings)에서 Pages를 활성화하여 최종 웹앱 도메인(`https://[username].github.io/[repo-name]`) 획득.

## 5. 개발 환경 및 도구 추천
* **IDE (통합 개발 환경):** Cursor AI (AI의 도움을 받아 Apps Script 코드 및 JavaScript 로직을 빠르게 자동 완성하고 오류를 수정하는 데 최적화됨).
* **버전 관리:** Git & GitHub Desktop (간편한 커밋 및 푸시).
* **테스트:** Chrome 브라우저 개발자 도구 (F12).

## 6. 향후 확장 가능성 (Optional)
* **관리자 기능:** 웹앱 내에 숨겨진 관리자 버튼을 두어, 클릭 시 원본 구글 시트로 바로 이동하여 링크를 수정/추가할 수 있는 단축키 마련.
* **디자인 고도화:** 학급 학생들도 함께 사용할 경우, 친근한 UI 디자인이나 학교/학급 로고 배치 고려.
